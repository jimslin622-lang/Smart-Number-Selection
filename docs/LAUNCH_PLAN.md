更新时间：2026-06-14

## 一、当前状态结论

当前代码已达到“本地可体验 / 可导入微信开发者工具预览”的阶段：

- 项目目录：`D:\AI zhushou\project`
- 页面、配置、图标、JS 依赖路径检查通过
- JSON 配置检查通过

- 当前数据层仍为本地模拟数据：`services/lottery-api.js`

但距离正式上线，还有一个关键合规分叉：

> 微信开放文档中，涉及“体育/福利数字查询、展示、线上生成”等服务，通常要求国家/地方数字管理中心等对应资质主体；且不支持在线销售。普通个人主体/普通企业主体直接以数字生成工具提审，存在较高被驳回风险。

## 二、上线前必须确认的问题

### 1. 小程序主体

需要确认小程序账号主体是：

- 个人主体
- 普通企业主体
- 数字管理中心/授权机构主体

如果是个人主体：不建议按“数字/生成/示例/使用”方向直接上线。

如果是普通企业主体：仍需核对是否具备数字相关服务类目所需资质。

如果是官方/授权主体：可以按数字信息工具方向准备材料，但仍不能涉及在线销售、代购、合买、返利、预测结果等内容。

### 2. 产品定位选择

上线有两条路：

#### 路线 A：保持数字工具定位

适合：具备相关主体资质或授权。

需要做：

- 确认服务类目能添加成功
- 提供真实示例数据来源
- 页面明确“不销售数字、不代购、不预测结果”
- 准备资质材料、数据来源说明、审核备注

风险：无资质时大概率被驳回。

#### 路线 B：转成智能选号 / 概率科普 / 娱乐工具

适合：个人主体或普通主体想尽快上线。

需要做：

- 去掉或弱化“数字、使用、示例、彩种、命中、实际使用”等强数字词
- 不展示真实彩种名称和历史开奖
- 将功能改成“随机号码生成、数字组合记录、概率科普”
- 文案统一为“娱乐随机工具，不对应任何实际实际使用场景”

风险：如果页面仍高度复刻数字玩法，也可能被认为规避数字类目。

## 四、当前代码中需要重点整改的风险点

### 高风险词/功能

当前页面中存在：

- “彩票选号助手”
- “彩票类型”
- “参考周期”
- “最新开奖”
- “历史开奖”
- “轻松使用”
- “高频数字/低频数字/分布”
- 多个真实彩种名：双色球、大乐透、福彩3D、排列3、排列5、快乐8、七乐彩、七星彩、香港六合等

如果走路线 A，这些词可以保留但要有资质。

如果走路线 B，这些词需要系统性替换/删除。

### 数据风险

当前 `services/lottery-api.js` 使用模拟数据，但页面展示形式接近真实示例数据。

建议：

- 路线 A：接真实合法数据源，并在页面说明数据来源。
- 路线 B：不要展示“示例/历史开奖”，改成“历史生成记录/示例数据”。
